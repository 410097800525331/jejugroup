# 하이브리드 구조 실행 체크리스트

이 문서는 `전면 React 전환` 이 아니라 `하이브리드 구조를 운영 가능한 최종 상태로 고정` 하기 위한 실행 체크리스트다

## 이번 라운드에서 끝낸 항목

- [x] `jejuair` 비전환 고정 정책을 운영 문서에 명시
- [x] `front` 단일 수정 원본 원칙을 하이브리드 기준으로 재정의
- [x] `apps/cs`, `apps/shell`, `components/react` 책임선을 운영 문서에 명시
- [x] 현재 엔트리포인트를 렌더링 모드 기준으로 분류한 인벤토리 문서 추가

## 다음으로 해야 할 항목

### 1. 구조 선언 고정

- [x] `AGENTS.md` 와 `docs/transition-architecture.md` 표현을 하이브리드 최종 기준으로 일치시킬 것
- [ ] `front -> jeju-web` 미러 운영이 계속 필요한 이유와 수정 금지 경계를 문서 한 곳에서 끝내도록 정리할 것

### 2. 엔트리포인트 책임 고정

- [ ] 각 엔트리포인트에 대해 `원본 위치`, `렌더 방식`, `수정 위치`, `배포 경로`를 유지할 것
- [ ] 새 페이지 추가 시 반드시 `고정 정적`, `하이브리드`, `독립 앱` 중 하나로 분류하고 시작할 것

### 3. 런타임 경계 정리

- [ ] `front/components/runtime/*` 와 `front/.generated/webapp-overlay/*` 를 수정 금지 산출물로 공식화할 것
- [ ] 개발 서버에서만 소스 직결되는 경로와 빌드 산출물 경로를 문서로 분리할 것

### 4. 백업 정책 고정

- [x] `front/backup` 과 `front/jejuair/backup` 을 제거하고 활성 문서에서 backup 전제를 걷어낼 것
- [x] backup 경로가 런타임 경로로 다시 연결되지 않도록 개발 서버 규칙과 인벤토리를 정리할 것

### 5. 테스트 범위 고정

- [ ] smoke 대상 엔트리를 하이브리드 기준으로 다시 점검할 것
- [ ] `cs` 독립 앱, `shell` 하이브리드 페이지, `jejuair` 고정 정적 페이지가 각각 최소 1개 이상 자동 검증되도록 유지할 것

### 6. 워크트리 위생

- [ ] 생성물과 원본 변경을 분리해서 커밋 단위를 다시 자를 것
- [ ] 임시 pid, 캡처 이미지, 실험 파일이 작업트리에 남지 않도록 정리할 것

## 완료 판정 기준

아래가 모두 만족되면 `하이브리드 구조 정리 완료` 로 본다

- [ ] 문서가 더 이상 `전환기 임시 체계` 라고 설명하지 않는다
- [ ] `jejuair` 비전환 고정 정책이 예외가 아니라 공식 규칙으로 남아 있다
- [ ] `front` 에서 어떤 페이지를 어디서 고쳐야 하는지 팀이 헷갈리지 않는다
- [ ] 배포 미러와 원본의 책임선이 문서와 스크립트에서 일치한다
- [x] backup 경로가 활성 repo layout 에서 제거되어 더 이상 운영 판단을 흐리지 않는다
